# 496. Next Greater Element I

## Problem Statement

The **next greater element** of some element `x` in an array is the first greater element that is to the **right** of `x` in the same array.

You are given two distinct 0-indexed integer arrays `nums1` and `nums2`, where `nums1` is a subset of `nums2`.

For each `0 <= i < nums1.length`, find the index `j` such that `nums1[i] == nums2[j]` and determine the **next greater element** of `nums2[j]` in `nums2`. If there is no next greater element, the answer for that query is `-1`.

Return an array `ans` of length `nums1.length` such that `ans[i]` is the next greater element as described above.

---

## Approach — Monotonic Stack + Hash Map

1. Traverse `nums2` left to right, maintaining a **monotonic decreasing stack**.
2. For every element `x` in `nums2`, pop all elements from the stack that are **smaller than `x`** — `x` is their next greater element. Store these mappings in a hash map `obj`.
3. After the traversal, any element still in the stack has **no next greater element**, so map them to `-1`.
4. For each element in `nums1`, look up its answer directly from `obj`.

---

## Complexity

| | |
|---|---|
| **Time Complexity** | O(n + m) — `n` = length of `nums2` (single pass + stack ops), `m` = length of `nums1` (lookup) |
| **Space Complexity** | O(n) — hash map and stack each hold at most `n` elements |

---

## Code

```javascript
var nextGreaterElement = function (nums1, nums2) {
    let obj = {};    // maps element -> its next greater element
    let stack = [];  // monotonic decreasing stack
    let result = [];

    // Build the next greater element map for all elements in nums2
    for (let x of nums2) {
        // x is the next greater element for everything smaller at the top of the stack
        while (stack.length && stack[stack.length - 1] < x) {
            obj[stack.pop()] = x;
        }
        stack.push(x);
    }

    // Remaining elements in stack have no next greater element
    while (stack.length) {
        obj[stack.pop()] = -1;
    }

    // Answer queries for nums1 using the precomputed map
    for (let i = 0; i < nums1.length; i++) {
        result.push(obj[nums1[i]]);
    }

    return result;
};
```

## Dry Run

**Input:**
```
nums1 = [4, 1, 2]
nums2 = [1, 3, 4, 2]
```

**Step 1 — Build next greater element map by traversing `nums2`:**

| Step | Current `x` | Stack (before) | Action | Stack (after) | `obj` |
|------|-------------|----------------|--------|---------------|-------|
| 1    | 1           | []             | Push 1 | [1]           | {}    |
| 2    | 3           | [1]            | 1 < 3 → pop, obj[1]=3; push 3 | [3] | {1:3} |
| 3    | 4           | [3]            | 3 < 4 → pop, obj[3]=4; push 4 | [4] | {1:3, 3:4} |
| 4    | 2           | [4]            | 4 > 2 → push 2 | [4, 2] | {1:3, 3:4} |

**Step 2 — Drain remaining stack (no next greater element):**

| Pop | `obj` update |
|-----|--------------|
| 2   | obj[2] = -1  |
| 4   | obj[4] = -1  |

**Final `obj`:** `{ 1: 3, 3: 4, 2: -1, 4: -1 }`

**Step 3 — Answer queries for `nums1 = [4, 1, 2]`:**

| `nums1[i]` | `obj[nums1[i]]` |
|------------|-----------------|
| 4          | -1              |
| 1          | 3               |
| 2          | -1              |

**Output:** `[-1, 3, -1]`
